## 🅰 길 A: Netlify Drop (가장 쉬움 · 5분)

회원가입 안 해도 즉시 URL 받을 수 있는 가장 빠른 방법.

### 단계

**1. Netlify Drop 페이지 열기**
브라우저로 이동: <https://app.netlify.com/drop>

**2. deploy 폴더를 드래그**
- Finder/Explorer에서 `deploy` 폴더를 통째로 잡고
- 브라우저 화면 중앙 큰 점선 박스로 **드래그 앤 드롭**
- 폴더 안 파일들이 자동 업로드 (약 30초)

**3. URL 발급 받음**
- 자동 URL 예: `https://wonderful-pasta-12345.netlify.app`
- 이 URL이 진짜 인터넷 주소
- 카톡·인스타·어디든 공유 가능

**4. (선택) 회원가입 → URL 변경**
- 이메일로 가입 (무료)
- Site settings → Change site name
- 원하는 이름으로 변경: 예) `andrew-ireland`
- 새 URL: `https://andrew-ireland.netlify.app`

**5. 카톡으로 친구들에게 공유**
```
"안녕하세요, 더블린 차량 동행 서비스 안내 페이지입니다."
🇮🇪 Beautiful Ireland with Everyone
https://andrew-ireland.netlify.app
```

### 나중에 사이트 업데이트하려면
- 회원가입 안 한 경우: 새로 드래그 (URL 매번 새로)
- 회원가입한 경우: 같은 사이트에 다시 드래그하면 동일 URL에서 업데이트

## 🅱 길 B: GitHub + GitHub Pages (영구 · 10분)

GitHub 계정으로 영구 무료 호스팅 + 버전 관리.

### 단계

**1. GitHub 가입** (없으면)
<https://github.com/signup> 가입 (무료)

**2. 새 Repository 생성**
- 우상단 + 버튼 → New repository
- Repository name: `andrew-ireland` (또는 원하는 이름)
- Public 선택
- ✅ "Add a README file" 체크
- Create repository 클릭

**3. 파일 업로드**
- 새 repo 메인 페이지에서 "Add file" → "Upload files"
- `deploy/` 폴더 안의 **모든 파일**을 드래그
  - `index.html`
  - `photo1.jpg`, `photo2.jpg`, `photo3.jpg`
  - `robots.txt`, `sitemap.xml`
  - (`README.md`는 이미 있으면 덮어쓰기)
- 페이지 아래 "Commit changes" 클릭

**4. GitHub Pages 활성화**
- 같은 repo의 Settings 탭
- 좌측 메뉴에서 "Pages"
- Source: "Deploy from a branch"
- Branch: `main` / Folder: `/ (root)`
- Save 클릭
- 약 1–3분 후 URL 발급: `https://[username].github.io/andrew-ireland/`

**5. 카톡으로 공유**
```
🇮🇪 Beautiful Ireland with Everyone
https://[username].github.io/andrew-ireland/
```

### 나중에 업데이트
- GitHub repo에서 파일 직접 수정 (또는 다시 업로드)
- 자동으로 사이트 반영 (1–3분)
